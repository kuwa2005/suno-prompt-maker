## 決め事・ルール

### 応答
- 応答は日本語で

### デザイン
- 確認ダイアログのボタン順は「はい」→「いいえ」（同意/非同意の順）
- Tailwind CSS CDN版を使用（CLI移行はレイアウト崩壊のため撤回済み）
- ダーク/ライトモード対応

### デプロイ
- **必ず** `npm run build` で `dist/` を生成してから FTP デプロイ
- FTPデプロイ: `./deploy-ftp.sh` — `dist/` をミラー転送（認証は `deploy.env` 推奨）
- デプロイ前に `npm run check`（または `node --check src/*.js`）で構文チェック
- `data.js` の `enka` と `enkaMusic` プロパティが正しくデプロイされているか確認

### カテゴリ管理
- カテゴリ追加は `src/data.js` の2箇所（CATEGORIES + PROMPT_DATA）
- `CATEGORIES` の `dataKey` は `PROMPT_DATA` のキーと一致させること
- 現在のカテゴリ数: **38**
- `data.js` 変更後は IndexedDB キャッシュの再構築（About タブ）を推奨

### AI機能
- **ternlight**: `@ternlight/base` を Vite + WASM プラグインでバンドル（本番は WASM モードが基本）
- WASM 失敗時はフォールバックモード（簡易類似度）。キャッシュキーは `wasm` / `fallback` で分離
- **プロンプト生成（ランダム）**: 従来の重み付きラウンドロビン
- **プロンプト生成(AI)**: メインプロンプト + スライダー重みに基づくセマンティックタグ選択（`ai-prompt-generator.js`）
- **(AI) メインプロンプトから反映**: `auto-setter.js`（`excludeGenre=false` でジャンルも含む）
- **IndexedDB**: `tag-index-db.js` が埋め込みベクトルを保存。初回のみインデックス構築
- Chrome Translator API: Chrome 138+ のみ対応、ユーザージェスチャーが必要（任意）

## 開発・デプロイ手順

```bash
cd /home/mimocode/workspace/suno-prompt-maker
npm install
npm run dev        # 開発
npm run build      # 本番ビルド → dist/
cp deploy.env.example deploy.env   # 初回のみ（FTP 情報を記入）
./deploy-ftp.sh    # FTP
```

## 注意事項

1. **ビルド必須**: ルートの `index.html` を直接本番に上げるのではなく、常に `dist/` をデプロイする
2. **data.js のデプロイ確認**: `enka` と `enkaMusic` が含まれるビルドか確認
3. **Tailwind CSS**: CDN版を使用。CLI版はレイアウト崩壊の原因となるため使用不可
4. **IndexedDB**: カテゴリ・タグを増減したら About のキャッシュ再構築を実行
5. **Playwright**: デバッグ用にセットアップ済み。`node debug.js` でコンソールログ取得可能
