# [S1] 概要

ternlight を活用した Suno Prompt Maker のローカル AI 機能統合スペック

## [S2] 背景

現在の Suno Prompt Maker は、ユーザーが手動でカテゴリの%を調整、またはランダム生成でプロンプトを作成。ternlight を組み込むことで、自然言語入力から自動で最適な設定を算出、タグ候補の意味ベース検索、過去履歴のあいまい検索を実現する。

## [S3] 技術選定

| 項目 | 選定 |
|------|------|
| ライブラリ | @ternlight/base (7MB, ~5ms/embed) |
| アプローチ | ternlight + ルールベース |
| 実装範囲 | ①自動セッティング ②セマンティック検索 ③履歴検索 |

## [S4] アーキテクチャ

### ファイル構成（新規追加）

```
suno-prompt-maker/
├── ternlight-engine.js    # ベクトル化・類似度計算ラッパー
├── auto-setter.js         # %自動算出ルール
├── tag-index.js           # タグベクトルキャッシュ管理
└── ...
```

### データフロー

```
ユーザー入力 → embed() → ベクトル → cosineSim() → 類似度スコア → ルール変換 → UI反映
```

## [S5] 機能①: 自動セッティング

### 要件
- ユーザーが「AIイメージ入力」エリアにテキストを入力
- 「AIで反映する」ボタンまたはEnterキーで実行
- 全タグとの類似度を計算し、上位カテゴリの%を自動設定

### ルール
- 類似度 > 0.3 のカテゴリのみ対象
- 類似度スコアを正規化して%に変換（合計100%）
- 残りは0%に設定
- スライダーにアニメーション付きで反映

### UI
- メインプロンプト直下に「AIイメージ入力」テキストエリアを新設
- プレースホルダーにサンプル入力を表示
- 反映後、調整されたスライダーを数秒間ハイライト

## [S6] 機能②: セマンティック検索

### 要件
- 各カテゴリのタグ候補にインクリメンタル検索機能を追加
- 入力テキストの意味に近いタグをリアルタイムでサジェスト

### UI
- 各カテゴリセクションの横に「🔍 候補を絞り込む」検索窓を追加
- keyup イベント毎にリアルタイムで並び替え
- 類似度低いタグは opacity: 0.4 で非表示化

### パフォーマンス
- 入力から表示まで < 10ms

## [S7] 機能③: 履歴のあいまい検索

### 要件
- 履歴タブに検索バーを追加
- 過去のプロンプトを意味ベースで検索可能に

### 実装
- localStorage の履歴エントリをベクトル化
- 初回アクセス時にのみベクトル化し、 sessionStorage にキャッシュ
- 類似度順にソートして表示

### 制限
- 履歴は最大50件（既存の制限）
- ベクトル化には約2.5ms/件

## [S8] パフォーマンス目標

| 操作 | 目標時間 |
|------|----------|
| WASM初期ロード | < 2秒 |
| タグベクトル化（全35カテゴリ） | < 3秒（初回） |
| セマンティック検索 | < 10ms |
| 自動セッティング | < 50ms |

## [S9] エラーハンドリング

- WASM ロード失敗: 通常の手動モードにフォールバック
- ベクトル化エラー: その入力をスキップし、空の結果を返す
- タイムアウト: 500ms でタイムアウトし、フォールバック

## [S10] テスト戦略

- 単体テスト: ternlight-engine.js のベクトル化・類似度計算
- 結合テスト: 自動セッティングのルールロジック
- E2Eテスト: ユーザー入力からUI反映まで
